# 시설 소개 문장 갱신 계획

## 목표

목록 카드와 지도 팝업에 들어가는 시설 소개는 부모가 방문 여부를 빠르게 판단할 수 있도록 작성한다. 모든 문장은 카드에 이미 표시되는 연령, 요금, 운영일, 주차 여부를 반복하지 않고, 그 시설만의 구체적인 특징을 최소 1개 포함한다.

## 작성 규칙

- 분량: 카드 본문 기준 70~110자, 지도 팝업에서도 말줄임 없이 읽히는 길이.
- 내용: 공간 특징, 대표 전시/자료실, 아이 동선, 체험 포인트, 주변 환경 중 1~2개만 선택.
- 제외: 카드의 별도 영역에 이미 표시되는 연령, 요금, 운영요일, 주차, 예약, 결제 방식, 준비물은 소개 문장에서 반복하지 않는다.
- 출처: 공식 홈페이지, 서울시/자치구 페이지, 네이버지도 소개, 블로그 후기 상위 결과를 우선 확인한다.
- 표현: 후기 문장을 그대로 옮기지 않고 부모 관점으로 요약한다.
- 금지: 별점, 평점, 과장 광고, 확인되지 않은 “최고/최대/인기” 표현.

## 1차: 박물관·체험

대상은 `data/museums.json` 전체다. 국립/시립/구립 기관은 공식 홈페이지를 우선 보고, 체험형 시설은 네이버지도 소개와 최근 후기 상위 결과에서 실제 관람 포인트를 확인한다.

검색 예시:

```text
{시설명} 어린이 체험 후기
{시설명} 가족 관람 후기
{시설명} 공식 홈페이지 전시 체험
```

품질 기준:

- 전시 주제만 쓰지 않고 아이가 실제로 볼 것 또는 할 것을 하나 넣는다.
- 청소년경찰학교처럼 같은 유형 시설이 여러 곳인 경우 공통 설명을 쓰되, 자치구명과 운영 주체를 구분한다.

## 2차: 도서관

대상은 `data/libraries.json` 전체다. 어린이 전문 도서관은 특화 공간을, 종합도서관은 어린이자료실/유아실/가족실의 구체적인 강점을 적는다.

검색 예시:

```text
{도서관명} 어린이자료실
{도서관명} 유아자료실 후기
{도서관명} 아이랑 도서관
```

품질 기준:

- “어린이 자료와 프로그램이 있다” 같은 일반 문장을 피한다.
- 한옥, 영어 특화, 마루형 열람실, 수유실, 숲 조망, 이야기방처럼 구별되는 시설 요소를 우선 반영한다.

## 3차: 키즈카페

대상은 `data/kids_cafes.json` 전체다. 시설별 놀이 구조, 예약/결제 주의점, 보호자 동선, 미끄럼방지 양말 등 방문 준비 정보를 확인한다.

검색 예시:

```text
서울형 키즈카페 {시설명} 후기
{자치구} {시설명} 키즈카페 예약 후기
{시설명} 우리동네키움포털
```

품질 기준:

- 같은 서울형 키즈카페라도 “실내 놀이공간”으로 통일하지 않는다.
- 사진과 후기에서 보이는 대표 놀이 요소가 확인되면 한 가지를 반영한다.

## 반복 갱신 주기

- 정기 갱신: 3개월마다 1차 → 2차 → 3차 순서로 반복.
- 수시 갱신: 공식 링크가 바뀌거나 시설 폐관/휴관/리모델링이 확인되면 즉시 수정.
- 기록 필드 권장: `summaryReviewedAt`, `summarySourceType`, `summarySourceNote`.

## 검수 체크리스트

- 시설마다 고유 특징이 최소 1개 있는가?
- 카드에 이미 보이는 요금/연령/운영일만 반복하지 않았는가?
- 지도 팝업에서 문장이 줄바꿈되어도 버튼과 겹치지 않는가?
- 공식 페이지와 네이버지도 링크가 모두 열리는가?
- 몇 달 뒤 다시 확인할 수 있도록 출처 유형을 남겼는가?
