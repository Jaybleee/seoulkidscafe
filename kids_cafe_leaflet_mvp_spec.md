# 서울형 키즈카페 지도 웹페이지 MVP 개발 명세

## 1. 목적
서울형 키즈카페 예약 사이트의 지점 정보를 보기 쉽게 재구성한 정적 웹페이지를 만든다.

목표는 다음과 같다.
- 지도에서 각 키즈카페 위치를 확인할 수 있어야 한다.
- 카드형 리스트로 지점을 한눈에 볼 수 있어야 한다.
- 지점 클릭 시 사진, 기본 정보, 예약 링크를 볼 수 있어야 한다.
- 무료로 배포 가능해야 한다.
- 비개발자도 이후 데이터를 갱신할 수 있어야 한다.

---

## 2. MVP 범위
### 포함
- 서울형 키즈카페 목록 크롤링
- 일반형 키즈카페 + 여기저기 키즈카페 동시 수집
- 이름, 주소, 전화번호, 이용연령, 이용정원, 예약 링크 수집
- 가능하면 썸네일 이미지 링크도 수집
- 이용안내 기준 주요 특징 요약 수집
- 토요일/일요일 운영 여부 수집
- 주차 가능 여부 및 주차 안내 수집
- 수집 데이터를 JSON 파일로 저장
- Leaflet + OpenStreetMap 기반 지도 웹페이지 구현
- 지도 마커 표시
- 카드형 리스트 표시
- 카드 클릭 시 해당 위치로 지도 이동
- 마커 클릭 시 팝업 표시
- 예약 페이지 이동 버튼 제공
- 자치구 텍스트 필터
- GitHub Pages 배포 가능 구조

### 제외
- 로그인 기능
- 예약 기능 직접 구현
- 서버/DB 구축
- 관리자 페이지
- 자동 스케줄 실행
- 모바일 앱 개발

## 3. 기술 구성
### 필수
- Python 3.x
- requests
- beautifulsoup4
- pandas
- Leaflet.js
- OpenStreetMap 타일
- HTML / CSS / JavaScript
- GitHub Pages

### 선택
- geopy 또는 별도 지오코딩 스크립트
- Playwright 또는 Selenium (정적 크롤링이 안 될 때만)

기본 원칙:
- 가능하면 정적 크롤링으로 처리한다.
- 브라우저 자동화 도구는 마지막 수단으로만 사용한다.
- 웹페이지는 서버 없이 동작하는 정적 파일 구조로 만든다.

## 6. 크롤러 요구사항
### 6.1 대상
- 서울형 키즈카페 목록 페이지
- 필요 시 상세 페이지 추가 접근

### 6.2 수집 항목
- 지점명
- 지점 유형(일반형/여기저기)
- 주소
- 전화번호
- 이용연령
- 이용정원
- 예약 링크
- 썸네일 또는 대표 이미지 링크
- 이용안내 기준 주요 특징
- 토요일 운영 여부
- 일요일 운영 여부
- 주차 가능 여부
- 주차 정보

### 6.3 동작 방식
1. 목록 페이지 요청
2. 페이지네이션 순회
3. 각 카드/목록에서 텍스트 추출
4. 예약 링크 추출
5. 이용안내 상세 페이지 접근
6. 주요 특징, 운영일, 주차 정보 추출
7. 가능하면 이미지 링크 추출
8. 중복 제거
9. CSV 저장
10. 주소 기반 위경도 보강
11. 최종 JSON 저장

### 6.4 예외 처리
- 일부 필드가 없으면 빈 문자열 허용
- 위경도 변환 실패 시 해당 항목은 로그 기록
- 예약 링크가 상대경로면 절대경로로 변환
- 같은 이름 + 주소 조합은 중복으로 간주

### 6.5 출력 파일
- `kids_cafes_raw.csv`: 1차 수집 결과
- `kids_cafes.csv`: 위경도 보강 결과
- `kids_cafes.json`: 웹페이지 입력용 최종 결과

---

## 7. 위경도 처리 요구사항
가장 단순한 방식으로 주소를 위경도로 변환한다.

우선순위:
1. Google Sheets에서 수동 보정 가능하게 주소 컬럼 유지
2. 파이썬에서 자동 지오코딩 시도
3. 실패 항목은 별도 로그로 저장

주의:
- 무료/공개 지오코딩은 요청 제한이 있을 수 있으므로 과도한 호출 금지
- 캐시 파일을 두어 재실행 시 중복 지오코딩 방지

권장 출력 추가 파일:
- `geocode_cache.json`
- `geocode_failures.csv`

## 8. 웹페이지 요구사항
### 8.1 레이아웃
데스크톱:
- 좌측: 지도
- 우측: 카드 리스트

모바일:
- 상단: 지도
- 하단: 카드 리스트

### 8.2 상단 UI
- 페이지 제목
- 검색 입력창
- 자치구 필터 드롭다운
- 결과 개수 표시

### 8.3 카드 UI
각 카드에는 아래 정보 표시
- 대표 사진 1장 (없으면 placeholder)
- 지점명
- 키즈카페 유형
- 자치구
- 주소
- 주요 특징 요약
- 이용연령
- 이용정원
- 토요일/일요일 운영 여부
- 주차 가능 여부
- 주차 안내
- 예약 버튼

### 8.4 지도 UI
- 모든 지점 마커 표시
- 마커 클릭 시 팝업 표시
- 팝업 내용:
  - 이름
  - 사진
  - 주소
  - 예약 링크 버튼
- 카드 클릭 시 해당 마커 위치로 이동하고 팝업 열기

### 8.5 필터 기능
- 검색어로 이름/주소 검색
- 자치구 필터
- 필터 결과에 따라 카드와 마커 동시 갱신

## 14. 실행 흐름
### 1단계: 데이터 수집
```bash
pip install -r crawler/requirements.txt
python crawler/crawler.py
```

### 2단계: 위경도 보강
```bash
python crawler/enrich_geocode.py
```

```bash
python -m http.server 8000
```

## 15. 품질 기준

## 16. 한계 및 후속 개선
### 현재 한계
- 원본 사이트 구조 변경 시 크롤러 수정 필요

- 무료 지오코딩 정확도 이슈 가능

- 현재 위치 기반 가까운 지점 정렬
- 즐겨찾기
- 자동 업데이트 스크립트
- 네이버지도/카카오지도 버전 별도 개발

## 17. 코덱스에 줄 작업 지시문
아래 요구를 만족하는 최소 동작 프로젝트를 생성한다.

- Python 크롤러로 서울형 키즈카페 목록을 수집한다.
- 결과를 CSV와 JSON으로 저장한다.
- Leaflet + OpenStreetMap 기반 정적 웹페이지를 만든다.
- 지도 좌측, 카드 리스트 우측 구조를 기본으로 한다.
- 카드 클릭 시 지도 이동 및 팝업이 열리게 한다.
- 검색과 자치구 필터를 제공한다.
- GitHub Pages에 배포 가능한 구조로 만든다.
- 코드에는 실행 방법과 데이터 갱신 방법을 README에 정리한다.
- 프레임워크 없이 HTML/CSS/Vanilla JS 중심으로 구현한다.
- 처음 실행해도 에러가 적도록 방어 코드를 포함한다.

## 18. 가장 중요한 개발 원칙
- 처음부터 완벽하게 만들지 않는다.
- 사진이 안 되면 사진 없이 먼저 완성한다.
- 지도 + 카드 + 예약 링크까지를 MVP 완료 기준으로 삼는다.
- 데이터 갱신이 쉬운 구조를 최우선으로 한다.
